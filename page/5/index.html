关于阿里云轻量级应用服务器中WordPress中将HTTP转换为HTTPS中踩的坑

发表于 2022-03-22 更新于 2023-03-20 分类于 前端知识

1.1 ssl 证书问题

环境为 Nginx，申请的为阿里云免费证书，在阿里云控制台中的 ssl 证书服务中的免费证书处申请。 因为是轻量级应用服务器，不能够直接用阿里云部署，所以需要自己手动部署。但是按照阿里云官方的 ssl 证书部署教程来的话，就会导致 WordPress 无法进入，而且就算修改了 nginx.conf 文件里的 location 还是不太行，会报 403/404/405 的错，因为俺也是新手，所以不是很懂为什么。 就是如果只开放 443 端口的话，就能通过服务器直接进入，但是网页不能够登入。开放 80 端口将 http 转成 https 就会导致两种方式都无法登录。 最后使用的是一种取巧的办法，因为在网络安全这里可以看到网站的协议是 https。 所以我想可不可以直接将证书植入到原有的 ssl 中，做一个替换操作。 所以在/usr/local/nginx/conf/ssl 中看到了示例用的 ssl 证书，再将这里面的内容进行替换。 但是阿里云下载的 Nginx 的 ssl 证书没有 crt 文件，所以又用腾讯云做了一下转换，在腾讯云上上传证书，再重新下载下来，就会有如图所示的三个文件，将这三个文件进行替换，http 就转换为 https 了。 当然这是一种取巧的办法，如果有更好的方法不建议这么做。

1.2 WordPress 问题

登录问题可以通过修改/data/wwwroot/wordpress/wp-config.php 文件来进行。 在@package WordPress下方加就能进行正常的 https 登录了。

图片显示问题可以在自身使用的主题里/data/wwwroot/wordpress/wp-content/themes/自身主题下的 functions.php 中最下面添加就可以解决。

最后在/data/wwwroot/wordpress/wp-includes/functions.php 中的第 8 行左右有一个require ABSPATH . WPINC . '/option.php';的下面添加完成设置。

最后在 WordPress 仪表盘中的设置页面将这两个改为 https 完成最终的设置。

VIT精读 Vision Transformer

发表于 2022-03-17 更新于 2023-03-20 分类于 计算机视觉

An IMAGE IS WORTH 16×16 WORDS: TRANSFORMERS FOR IMAGE RECOGNITON AT SCALE 把一个图像分割成很多个大小为16×16的patch，再用transformer做大规模的图像识别

1 摘要

通常自注意力机制都是和CNN结合的，或者是attention去替换CNN的组件，但总体保持的还是CNN的结构，但是其实这是不必要的，这里说将attention和transformer一起用，效果特别好！！

2 引言

transformer在nlp中应用的时候，用大规模数据集去预训练，再在特定的领域的小数据集做微调。 在cv领域中，像素点输入的话，序列太长了，比如一个224×224的图片，转换成像素点序列就会有50176的长度，是长度为512的BERT的100倍。 VIT想直接用一个标准的transformer，少做修改，但是为了解决序列长度的问题，所以把一个图片打成了很多16×16的patch，224/16=14,将每一个patch当成一个元素，所以序列长度就是14×14=196，这对于transformer来说是一个还可以的长度，之后这些patch每一个都会经过FC layer（fully connected layer）得到了一个linear embedding，再将这个linear embedding输入到transformer中。 其实这里等于是用NLP的思想去做VISION，每一个patch相当于一个word，这也是标题的由来 但是VIT用的是有监督训练，NLP一般都是无监督的。 在中等大小的数据集中（比如imagenet），比resnet还要低几个点。但是这个是可以预期的，因为CNN有归纳偏置，也就是先验知识或者说是提前假设，而transformer没有先验知识，他只能从图片数据里自己学习对视觉的感知 但是当VIT在大数据集上进行预训练时，比如ImageNet21k（1400万）-JFT-300（3000万张）数据集上预训练，结果就更好了。

网页主要分为三部分： 结构（HTML） 表现（CSS）——层叠样式表 行为（JavaScript）

1.1 CSS基础

通过CSS可以分别为网页的每一层设置样式

  1. （行内样式表）在标签内部通过style属性来设置元素的样式(不推荐使用)，只对一个标签生效，的确有点蠢-.- <p style="color:red; font-size: 60px;"> HELLO WORLD </p>

  2. （内部样式表）将样式编写到head中的style标签中，通过CSS的选择器来选中元素并为其设置各种样式,只能对一个网页起作用，不能跨页面进行复用

  3. （外部样式表） 将CSS样式直接写到CSS文件中,然后通过link标签引入外部CSS文件(head里) <link rel="stylesheet" href="./style.css"> 这种方式可以使用浏览器的缓存机制，从而加快网页的加载速度
